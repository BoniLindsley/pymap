pymap.bytes¶

Defines useful types and utilities for working with bytestrings.

pymap.bytes.MaybeBytes = typing.Union[typing.ByteString, typing.SupportsBytes]¶

A bytes object or an object with a __bytes__ method.

pymap.bytes.MaybeBytesT = ~MaybeBytesT¶

A type variable bound to MaybeBytes.

pymap.bytes.as_memoryview(val)¶

Wrap a bytestring in a memoryview for performance, while maintaining bytes type annotation.

Parameters:val (bytes) – The bytestring to wrap in a memoryview.

Note

This is necessary because even though the ByteString docs state that bytes may be used to annotate memoryview objects, it does not appear to work in practice. If/when this is resolved, remove this function in factor of a simple ... = memoryview(val) call.

See also

python/mypy#4871

Return type:bytes
class pymap.bytes.BytesFormat(how)¶

Bases: object

Helper utility for performing formatting operations that produce bytestrings. While similar to the builtin formatting and join operations, this class intends to provide cleaner typing.

Parameters:how (bytes) – The formatting string or join delimiter to use.
format(data)¶

String interpolation into the format string.

Parameters:data (Iterable[Union[Bytestring, Supportsbytes]]) – The data interpolated into the format string.

Examples

BytesFormat(b'Hello, %b!') % b'World'
BytesFormat(b'%b, %b!') % (b'Hello', b'World')
Return type:bytes
join(*data)¶

Iterable join on a delimiter.

Parameters:data (Iterable[Union[Bytestring, Supportsbytes]]) – Iterable of items to join.

Examples

BytesFormat(b' ').join([b'one', b'two', b'three'])

class pymap.bytes.ReView(pattern)¶

Bases: object

Wrap a Pattern so that it takes memoryview strings.

This behavior is supported by re, but type stubs do not allow it. If that is ever resolved, remove this class.

Parameters:pattern (Pattern[bytes]) – The pattern to wrap.
