Bases: object

Parameters used and passed around among the parse() methods.

Parameters

  • continuations (Optional[List[memoryview]]) – The continuation buffers remaining for parsing.

  • expected (Optional[Sequence[Type[Parseable[~ParseableT]]]]) – The types that are expected in the next parsed object.

  • list_expected (Optional[Sequence[Type[Parseable[~ParseableT]]]]) – The types that are expect in a parsed list.

  • command_name (Optional[bytes]) – The name of the command currently being parsed, if any.

  • uid (bool) – The next parsed command came after a UID command.

  • charset (Optional[str]) – Strings should be decoded using this character set.

  • tag (Optional[bytes]) – The next parsed command uses this tag bytestring.

  • max_append_len (Optional[int]) – The maximum allowed length of the message body to an APPEND command.

  • allow_continuations (bool) – Allow literal strings that require continuation data.

Bases: pymap.parsing.command.CommandSelect

The IDLE command waits for the continuation string DONE from the client. During this wait, the server may be sending untagged responses indicating concurrent updates to the mailbox.

Parsing this command is a special case. The continuation string DONE is actually parsed by the parse_done() method. The parse() only parses the IDLE <CRLF> portion, and does not raise a RequiresContinuation exception.
