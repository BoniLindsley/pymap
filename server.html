pymap.server¶

Defines socket interaction, accepting connections and handling a basic command/response flow.

class pymap.server.IMAPServer(login, config)¶

Callable object that creates and runs IMAPConnection objects when asyncio.start_server() receives a new connection.

Parameters:
  • login (LoginProtocol) – Login callback that takes authentication credentials and returns a SessionInterface object.
  • config (IMAPConfig) – Settings to use for the IMAP server.
class pymap.server.IMAPConnection(commands, config, reader, writer)¶

Runs a single IMAP connection from start to finish.

Parameters:
  • commands (Commands) – Defines the IMAP commands available to the connection.
  • config (IMAPConfig) – Settings to use for the IMAP connection.
  • reader (StreamReader) – The input stream for the socket.
  • writer (StreamWriter) – The output stream for the socket.
class pymap.server.IMAPConfig(args, debug=False, executor=None, ssl_context=None, starttls_enabled=True, reject_insecure_auth=True, preauth_credentials=None, max_append_len=1000000000, bad_command_limit=5, disable_idle=False, **extra)¶

Configurable settings that control how the IMAP server operates and what extensions it supports.

Parameters:
  • args (Namespace) – The command-line arguments.
  • debug (bool) – If true, prints all socket activity to stdout.
  • executor (Optional[Executor]) – If given, all backend operations will be run inside this executor object, e.g. a thread pool.
  • ssl_context (Optional[SSLContext]) – SSL context that will be used for opportunistic TLS. Alternatively, you can pass extra arguments cert_file and key_file and an SSL context will be created.
  • starttls_enabled (bool) – True if opportunistic TLS should be supported.
  • reject_insecure_auth (bool) – True if authentication mechanisms that transmit credentials in cleartext should be rejected on non-encrypted transports.
  • preauth_credentials (Optional[AuthenticationCredentials]) – If given, clients will pre-authenticate on connection using these credentials.
  • max_append_len (Optional[int]) – The maximum allowed length of the message body to an APPEND command.
  • bad_command_limit (Optional[int]) – The number of consecutive commands received from the client with parsing errors before the client is disconnected.
  • disable_idle (bool) – Disable the IDLE capability.
  • extra (Any) – Additional keywords used for special circumstances.
